AI 应用最重要的事情

有价值的 AI 应用只有两个标的
要么好玩 要么有用

前者的典型是 AI 陪伴
后者的典型是 Cursor

深入讨论一下所谓“有用”
无外乎几个点
省时间 简化流程
效率提高十倍

深入效率
对于开创自己事业的“创始人们” 效率就是生命
对于老板来说 效率就是降本增效 因为 Agent 可以取代部分人力

但对于打工人来说
效率意味着让自己有更多的摸鱼时间
而不是完成更多的工作
因为工作完成的更快了便会有更多工作，无休止的工作，那不如慢慢折腾

再往下深挖一步
为什么要提高效率？
创始人为了做成自己的公司 然后挣钱
打工人为了满足上级的要求 然后拿工资
最终都会涉及到“挣到钱”这个点上
因为如果提高了效率但是没有挣到更多的钱
那么这个“效率”实际上是虚假的空转的

那么这样来看主打“有用”的 AI 工具的效率
对于创始人们，从零到一阶段的创始人，需要通过观点驱动判断做的东西是否有市场，所以能通过 Cursor 等工具加快验证，合理
从一到一百阶段的创始人，用 Lovart 做营销，用 Cursor 迭代代码，走在正确的路上，能更快的推进，合理
对于打工人，用 Cursor，用 Lovart，没有挣到更多的显性的钱，因为没有直面市场，效率与收益被公司隔断了，不合理
使用了能提高效率的工具，缺没有享受到对应的收益，极其不合理

核心问题在于，他们没有更多可以出售的时间了
他们的时间已经被公司买断
他们没有多余时间做出更多的探索、试错
于是日复一日深陷其中

如果他们可以有一个分身，在平行时空中，利用他们的储备，不论是知识，能力，还是其他
去和他们合伙挣钱，用最少的事情开创自己的收入管道
把一份时间出售多次 是多么诱人的商业模式

那么有什么事情，是每个人都可以做，并且每个人都可以挣到钱的基础？
人生故事，经历
有言道：“世界上的一切问题都有答案，你只要找到有答案的那个人”
AI 带来的是平均答案 AI 永远没办法取代经历的价值

举个例子
在保研季，小冰同学在准备保研面试
他们会上网查找大量“经验贴”
然后自己总结 或许会利用一下 AI 比较 E 的会直接私信去咨询
在这里一切 AI 都会黯然失色，因为再厉害的 AI 也只会给出 bullet 的一条条不会错的指导
或许 AI 可以搜索互联网的上下文，得到一些比平均更出色的内容

但互联网充斥着垃圾，人还在利用 AI 向互联网灌输越来越多的垃圾，AI 公司由反过来用这些垃圾训练 AI
不仅如此
每一个平台都是一个信息孤岛 让价值更加隐匿，只为服务公司利益，而非创作者利益
这里就是价值互联网的使命所在

让创作者直接创作 获得收益 而不依赖任何一个平台
尽可能让自己的价值暴露在所有有需要的人面前

如果小冰同学面前有这样一款产品
打开页面 是一个 Agent 的对话框 这个 Agent 是小冰同学的分身 有着他们的共同记忆 我们叫做大兵
小冰同学说“我马上要面试复旦 EMF 的夏令营了”
大兵就会在价值互联网上去找到去年申请成功的学姐，就读两年后已经毕业工作的学长，甚至直接找到了面试官老师
他们都部署了自己的 Agent 在互联网上，去提供有价值的信息

小冰向先和申请成功的学姐聊聊
但学姐不在线 Agent 知道学姐过去申请的经历
于是小冰直接向 Agent 付费聊天
简单聊了两句觉得非常愉快 于是预定了学姐的付费咨询
然后付费，约定好了时间，学姐告诉了她很多当时的经验和细节

小冰获得了很多知识 于是可以更充分的去准备面试复旦了
学姐也获得了被动收益，让 Agent 帮她挣钱，链接资源，提供价值

无数个小冰和学姐，在这里创建了他们的“大兵”Agent，用货币获取信息，用信息获取货币。
这就构成了价值互联网的基础。

每个人直接拥有价值的收益权。

第二步，再加上一些金融玩法
发行个人股票，共享收益分红，可转让，可打包
深度结合 Defi 的概念 让有价值的真实的信息被市场所筛选
实现信息的市场定价 而非坐地起价
RWA for Information 这是每个人都可以参与的 RWA

广告带货的商业模式同样可以嵌入
但必须是在金融模式保护下的广告嵌入
否则会泛滥
广告商可以向 Agent 嵌入软广 在有用户触发对应问题时 Agent 将会用合适的方式去推荐产品
这将直接为 Agent 主产生收益 无论是按被浏览次数还是其他


几个问题
1. 真实性：怎么证明经历不是伪造的？
2. 定价机制：谁来定价？
3. 隐私安全：个人经历是否涉及隐私问题？
4. 冷启动：早期如何起飞？

但这些不都是最一开始要面临的问题，却都要是想清楚的问题。
这是后面一定会碰到的，scalable 的问题。

然后根据目标倒推执行策略

藏宝图
整个创业的旅程就像先画出一张地图
先标记藏宝图的位置，有近处的藏宝图也有远处的藏宝图
然后根据当下所处的坐标以及要去的地方，倒推出行动的路线
有的时候要跨过河流，需要木筏
有的时候需要跨过江海，则需要轮船
提前配齐资源
能看清每个节点需要什么
在出发之时就配齐，然后稳步向前

前三个月的时间，
肯定是做出一个完整闭环的 V1 代产品，可能只需要一个月时间
然后拿着产品的 V1 去验证 Product-Market Fit
但是因为冷启动的时候没有办法让大家都获得价值（网络属性，只有节点多了，网络的价值才会指数增长）
所以前期专打一个聚焦的领域，例如留学 DIY 的用户（这个时间点比较合适）
一方面是部署方，一方面是准备 DIY 的学生
如果学生都愿意付费了，且这一套流程可以跑的通，那么 PMF 算是验证到了
这个时候还不需要金融属性，只需要最简单的方式去做验证
例如直接找到对应的一堆人，手工录入信息，用 unscalable 的方式
然后再去找使用他们的用户

六个月的时间
在一个市场上站稳脚跟之后
就可以迁移至其他市场，例如春招，这里做的是某一些市场的聚焦
然后这里要实现泛化
意味着非我们主攻引导的领域的交易同步发生，并且这一点更重要
这里的人是各色各样的，希望让自己的时间产生价值的人，于是愿意创建自己的分身
并且要让他们只要创建了分身就都能有收益，这样才可持续
所以在这个阶段，是聚焦+泛化两个目标
这里需要探索融资和币圈的拓展了

OKR 目标
Make something people want.


0-3 个月
Objective：做出 V1 产品并验证 PMF
- KR1：上限 V1 产品，完成 MVP（Agent 分身 + 对话 + 预约咨询 + 支付）
- KR2：获取 >= 500 注册用户，其中 >= 50 付费用户
- KR3：上线 >= 50个分身，覆盖目标市场（留学 DIY）
- KR4：实现 >= 5w GMV，注册到付费转化率 >= 10%
- KR5：首次付费用户 30 日留存 >= 40%

6 个月
Objective：在一个市场站稳脚跟，并启动泛化生态，让分身真正成为被动收益管道
- KR1：累计 >= 1w 注册用户，其中 >= 1000 付费用户
- KR2：累计 >= 1000 个分身，覆盖至少两个市场（留学 + 春招）
- KR3：月 GMV >= 50w，平台抽佣收入 >= 15w
- KR4：>= 20% 的交易来自非垂直市场（证明泛化）
- KR5： >= 30% 的分身创作者获得收益，平均分身月收益 >= 3000 元

1 年
Objective: 拓展多市场，跑通多元商业模式，开辟链上链路
- KR1：覆盖 5 个垂直市场
- KR2：累计 >= 10w 注册用户，其中 >= 1w 付费用户
- KR3：累计 >= 1w 个分身
- KR4： 年 GMV >= 5000w 元，平台抽佣 >= 1500w 元，实现盈利
- KR5：拓展广告收入 >= 20%
- KR6：链上平行平台启动，20% 收益来自于链上

3 年
Objective：形成价值互联网的雏形，成为高质量信息交易的核心入口
- KR1：累计 >= 100w 注册用户，其中 >= 10w 付费用户
- KR2：累计 >= 10w 个分身，涵盖全方位市场
- KR3：年 GMV >= 5 亿，平台抽佣 >= 1.5 亿
- KR4：>= 30% 的交易通过链上完成
- KR5：创作者平均月收益 >= 1w
- KR6：NPS用户推荐值 >= 50

10 年
Objective：实现完整的价值互联网，推动人类从“注意力互联网”进化到“价值互联网”
- KR1：全球用户 >= 1 亿，其中 >= 1000 万付费用户
- KR2：累计 >= 1000 万个分身，成为人类的数字资产网络
- KR3：年 GMV >= 1000 亿，平台抽佣 >= 300 亿
- KR4：信息资产总市值 >= 5000 亿元，确立新型信息金融市场
